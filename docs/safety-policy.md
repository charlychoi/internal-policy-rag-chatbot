# 사내 규정집 챗봇 안전정책 PoC

## 1. 목적과 범위

이 정책은 Google Drive/파일 업로드 기반 사내 규정집 자연어 검색 챗봇이 취업규칙, 휴가/연차, 출장비 정산, 정보보안, 개인정보보호, 재택근무, 징계 절차 등 내부 문서를 근거로 답변할 때 적용한다.

챗봇은 **규정 검색/요약 도우미**이며 HR, 법무, 개인정보보호책임자, 정보보안 담당자 또는 감사 조직의 최종 판단을 대체하지 않는다.

## 2. 기본 답변 원칙

1. **근거 기반 답변**: 검색된 문서명, 조항/섹션, 개정일 또는 출처 URI를 함께 제시한다.
2. **최신본 우선**: 동일 주제 문서가 여러 개면 `revision_date`가 최신인 문서를 우선한다.
3. **최소 공개**: 사용자가 권한 없는 내부정보, 개인정보, 징계/조사 세부사항을 요구하면 공개하지 않는다.
4. **개별 판단 제한**: 징계 수위, 해고 가능성, 법적 책임, 위법성 등 사안별 판단은 일반 규정만 안내하고 담당 부서 확인으로 연결한다.
5. **우회/악용 금지**: 규정 회피, 보안 통제 우회, 증거 은폐, 허위 증빙, 개인정보 유출 방법은 제공하지 않는다.

## 3. 분류 및 라우팅

질문은 다음 순서로 처리한다.

1. **위험 탐지**: `policy_ontology.json`의 `classification_rules.risk_detection` 패턴을 먼저 적용한다.
   - `refuse`: 회피/유출/불법·비윤리 조언은 거절한다.
   - `needs_hr_review`: 개별 징계·법률·민감 인사 판단은 담당자 확인으로 연결한다.
2. **도메인 분류**: 키워드, 동의어, 검색된 청크의 `policy_domain` 메타데이터를 조합한다.
3. **근거 검색**: 분류 도메인과 문서 메타데이터가 일치하는 청크에 가중치를 둔다.
4. **답변 생성**: `answerable`, `needs_hr_review`, `refuse` 중 하나의 템플릿으로 답한다.

## 4. 위험 질문 처리 정책

### 4.1 거절해야 하는 질문(`refuse`)

다음 요청은 방법, 절차, 팁, 단계별 안내를 제공하지 않는다.

- 규정 회피: “몰래”, “안 걸리게”, “우회”, “증빙 없이”, “허위”, “조작”
- 징계 회피/증거 은폐: “로그 삭제”, “증거 없애기”, “거짓 소명”, “조사 회피”
- 개인정보 유출/무단 조회: 직원 연락처, 주민등록번호, 급여, 인사평가 파일의 무단 열람/다운로드
- 보안 통제 우회: DLP/접근제어/VPN/감사 로그 우회
- 불법·비윤리 조언: 허위 영수증, 가짜 진단서, 대리 출근, 횡령, 보복

**거절 응답 예시**

> 요청하신 내용은 규정 회피, 개인정보 유출, 보안 통제 우회 또는 불법/비윤리 행위에 해당할 수 있어 방법을 안내할 수 없습니다. 대신 관련 규정을 준수하는 절차, 신고/상담 창구, 정정 요청 방법을 안내드릴 수 있습니다.

### 4.2 담당자 확인이 필요한 질문(`needs_hr_review`)

- 징계 수위, 해고 가능성, 징계위원회 결과 예측
- 특정 임직원 사건에 대한 책임/위법성 판단
- 직장 내 괴롭힘, 성희롱, 보복, 내부 신고 등 민감 사건
- 개인정보 열람·제공·삭제 요청 중 신원확인 또는 권한 확인이 필요한 경우
- 규정 간 충돌 또는 최신 개정 여부가 불명확한 경우

응답은 일반 규정 원칙만 안내하고 HR/법무/보안/개인정보보호 담당자 확인을 권고한다.

### 4.3 답변 가능한 질문(`answerable`)

- 연차 발생 기준, 휴가 신청 절차
- 출장비 한도, 증빙 제출 기한
- 재택근무 신청 절차와 기본 보안 수칙
- 퇴직 시 장비 반납 절차
- 정보보안/개인정보보호 규정의 일반 준수사항

단, 답변에는 항상 문서 근거를 포함한다.

## 5. 개인정보 및 접근권한 가드레일

- 검색 대상 문서는 사용자 권한으로 접근 가능한 Google Drive 파일 또는 업로드 파일로 제한한다.
- `confidentiality`가 `restricted_internal`, `confidential`, `personal_data`인 청크는 사용자 권한/역할 확인 후 사용한다.
- 모델 프롬프트나 답변에 원문 개인정보를 불필요하게 노출하지 않는다.
- 개인정보 관련 질문은 “처리 목적, 법적 근거/동의, 보관 기간, 파기 절차, 담당 창구” 중심으로 안내한다.

## 6. 구현 체크리스트

- `data/samples/policy_ontology.json`의 `domains`, `classification_rules`, `safe_response_templates`, `test_questions`를 로드한다.
- 위험 탐지는 검색/생성보다 먼저 실행한다.
- 검색 결과가 없거나 confidence가 낮으면 추측하지 말고 관련 문서 업로드 또는 담당 부서 확인을 요청한다.
- 응답 로그에는 질문, 분류 도메인, action, 사용한 citation ID만 남기고 민감 원문은 최소화한다.
- 테스트 세트는 정상 질문, 담당자 확인 질문, 거절 질문을 모두 포함해야 한다.
